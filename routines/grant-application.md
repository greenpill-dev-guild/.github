# Grant Application

> How the guild approaches grant applications — Octant, Gitcoin, Giveth, NLnet, and others.

**When to use this**: You are drafting a grant application on behalf of the guild or a guild project, or coordinating multiple applications across a grant cycle.

## Why we have a routine

Grant applications are high-leverage but easy to do badly. The guild is both **grantee and grantmaker**, which gives us pattern-matching most applicants don't have. Capture that here.

## Before you start

Decide:

- **Which grant?** Octant epochs, Gitcoin rounds, Giveth, NLnet, EF, project-specific funds. Each has different evaluation criteria — check the round-specific guidance first.
- **What's the scope?** A specific project (e.g. green-goods) or guild-wide ops? They are written differently.
- **What's the ask?** Fixed-amount or quadratic-funding-driven? Time-bound or ongoing?
- **Who decides?** Stewards approve guild-level applications. Project leads can drive project-specific applications, but stewards review before submission.

## Drafting flow

### 1. Read the round criteria carefully

What is the funder explicitly looking for? What evaluation framework do they use? Past funded projects are the best signal — read 3 of them.

### 2. Lead with the problem, not the solution

The strongest applications open with the **regenerative or coordination problem** the work addresses, not with what you're building. Funders see thousands of "we built X" pitches; they fund projects that articulate why the world needs X.

### 3. Show traction, not promises

For each claim, link to evidence:

- Live code → GitHub repo
- Active community → Telegram/Discord member counts and weekly call attendance
- Past funding → Octant Epoch 5, Giveth contributors, etc.
- Shipped work → green-goods pilots, cookie-jar deployments, GreenWill reputation, etc.

### 4. Use the guild's verified positioning language

The guild's mission language is consistent across all sources and proven over time. Reuse it:

- "A community of regen builders supporting Greenpill Chapters and regen communities."
- Mission triad: **impact measurement, capital allocation, community coordination**
- Values: **respect & inclusivity, transparency, integrity, collaboration, ethical development**
- "Building regenerative digital infrastructure" / "public goods"

Avoid:

- Growth-hacking language (FOMO, virality, leaderboards, streaks — these conflict with guild voice)
- Vague claims ("we will revolutionize X") without specifics
- "AI-first" or trendy-tech framing — funders evaluating regen work want substance

### 5. Be specific about the deliverables

Funders fund **deliverables**, not aspirations. For each ask:

- What will exist that doesn't exist now?
- How will success be measured?
- What's the timeline?
- What happens if you don't get the grant?

### 6. Get a steward review before submission

At least 24 hours before deadline. Stewards check:

- Is the ask appropriate for this funder?
- Is the language consistent with guild voice?
- Are the deliverables and budget honest?
- Does this conflict with another active application?

### 7. Submit and announce

After submission:

- Note the application in the forum and the relevant Discord channel
- Add or update the grant lifecycle item in Linear using the `funding:*` labels and saved funding views
- Set a calendar reminder to follow up if no response by their stated review window

## After award

If funded:

- Acknowledge publicly (Twitter, forum post)
- Update the project README with the funder badge / link
- Track deliverables against milestones; report progress to the funder per their cadence
- Include in quarterly overview

If declined:

- Request feedback if the funder offers it
- Note what you'd change for next round
- Don't reapply with the same pitch — iterate first

## Common pitfalls

- **Submitting at the last minute** — no time for steward review, no time to fix mistakes
- **Copying language across applications without adapting** — funders notice
- **Underclaiming or overclaiming** — be honest about traction; both extremes lose
- **Ignoring round-specific criteria** — generic pitches lose to specific ones
- **Forgetting to follow through** — funded grants without delivery damage future applications
